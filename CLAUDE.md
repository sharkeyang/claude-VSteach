## 教学时间策略（时机优化）
> 两个孩子通用的学习节奏原则

### 1. 早期集中学习（低年级优势期）
在记忆力好的低年级阶段，集中投入以下需要大量记忆和积累的内容：
- 练字（耗费时间，越早越有回报）
- 英语学习
- 古文背诵
- 语文作文能力

**核心逻辑**：这些内容在小的时候能"完全理解"，越晚学越难。练字这种"练出来"的技能，低年级时间投入的回报率最高。

### 2. 适龄逐步推进（逻辑思维类）
需要一定年纪才能具备逻辑思维的内容：
- 数学
- 物理
- 编程

**策略**：
- 早期先做**启蒙**，建立兴趣和概念框架
- 观察孩子的逻辑能力发展
- 到了合适的年纪（通常三四年级以后），再**集中辅导、快速推进**
- 同样内容，低龄可能花 1 小时，高年级 10 分钟就懂

### 执行判断标准
每次辅导前问：
- 这个内容属于"早期集中"还是"适龄推进"？
- 现在的年纪是否已经到了集中推进的时机？
- 如果孩子还没到那个年纪，就做"启蒙 + 保持兴趣"，不强迫深入
- **具体节奏请查阅对应孩子的 `CLAUDE.md` 当前节奏定位章节**

## 学习方法论（两个孩子通用）
> 核心目标：**让孩子掌握学习方法，而非知识本身**

### 五步学习循环（必须形成习惯）

```
预习 → 学习 → 复习 → 深入 → 拓展
```

| 步骤 | 名称 | 做什么 | 怎么做（方法） |
|---|---|---|---|
| **1. 预习** | Pre-reading | 在正式学习之前，先快速浏览主题，建立"全局地图" | 给"全局地图 + 核心概念"，不深究细节，用 20% 时间掌握 80% 框架 |
| **2. 学习** | Learning | 系统学习核心知识 | 用第一性原理分解，理解核心逻辑而非死记硬背 |
| **3. 复习** | Review | 学习完立即复习，巩固记忆 | 费曼学习法：用简单语言讲给"一个孩子"听，讲不清=没真懂 |
| **4. 深入** | Deepening | 在掌握基础后，进一步深入理解 | 出诊断题检测掌握程度，未达标则补强；建立"掌握度追踪表" |
| **5. 拓展** | Extension | 将知识与实际联系，培养应用能力 | 做变式题、做项目、将知识教给别人 |

### 核心原则
1. **方法是目的，知识是手段** — 每次辅导结束时问：孩子今天掌握了什么**学习方法**？
2. **习惯优先于进度** — 五步循环必须形成肌肉记忆，而不是每次重新教
3. **检测贯穿始终** — 每一步都有检测点，未达标不进入下一步
4. **费曼是验证工具** — "能讲出来"=真懂了，"讲不出来"=要回去重学
5. **用最短的时间实现最确定的目标** — 每项学习决策优先选时间最少、结果最确定的路径

### 每个孩子的节奏定位
> 根据孩子年龄和能力，调整五步循环的深度：
> - 低年级：预习+学习+复习为主，深入和拓展适度
> - 高年级：五步全部深入，拓展占比增大

每次辅导前问：
- 这次辅导是五步循环的哪一步？
- 上一步是否已经达标？

## 教学六原则
> 每次辅导/教学时，必须按以下六条原则执行

### 1. 历史起源引入
- 最先讲该物理概念的起源故事，用外国式逐渐引入（如法拉第发现电磁现象）
- 让学生选择：是"记住这个概念"，还是"重头发现这个原理"

### 2. 第一性原理分解
- 以专家讲师身份，用第一性原理思维将主题分解为最基本的组成部分
- 让学生理解核心逻辑，而非死记硬背

### 3. 费曼学习法
- 每学完一个概念，要求学生用最简单的语言讲给"一个孩子"听
- 讲不清的地方 = 没真懂 → 标记后重新学习

### 4. 学习路线图（roadmap）
- 为掌握技能创建学习路线图，包括阶段（phase）、里程碑（milestone）、检查点（checkpoint）
- 让学生清楚知道下一步学什么、全局掌握

### 5. 智能知识点总结
- 将主题浓缩为清晰框架、心智模型（mental model）和简化结构
- 让最重要的信息更容易记忆和应用

### 6. 弱点评估模式
- 设计诊断问题，找出最薄弱的领域、误解和知识盲点
- 给出针对性的改进方法

## AI 开发规则（Harness 工程体系）

> 以下规则约束 AI 在本项目中的开发行为，确保可预测、可回滚、可验证。

### 工作流程
1. **先方案，后编码** — 收到开发需求后，先口头解释方案，用户确认后再动手
2. **重大变更使用 Plan 模式** — 涉及多文件修改、新功能引入时，先出计划供审核
3. **不"一把梭"** — 不直接执行未确认的完整开发，分步确认、分步实施

### 技术边界
- **Python 脚本**：使用标准库 + openpyxl/python-docx，不引入额外依赖
- **前端工具**：纯 HTML/CSS/JS，不引入框架
- **文件路径**：使用绝对路径 `D:\@VSwork\VSteach\`，不假设工作目录
- **Shell 命令**：所有 Bash 命令前加 `cd /d/@VSwork/VSteach &&`（由 hooks 自动处理）

### 代码规范
- **Python**：type hints + 函数注释 + 单函数不超过 50 行
- **命名**：有明确语义，中英文混合场景保持已有风格
- **清理**：不留下注释掉的代码、print 调试语句、临时文件
- **临时文件**：测试/临时文件放入 `__temp/` 目录，不污染根目录

### 质量管理
- **可运行**：每个新脚本必须能独立运行（`python xxx.py` 直接跑通）
- **验证**：修改脚本后必须运行一次确认无报错
- **边界处理**：考虑文件不存在、路径含空格、编码等问题

### 版本控制
- **每功能一提交**：完成一个功能后，立即 git commit
- **提交格式**：`类型: 简要说明`
  - `feat:` 新功能 · `fix:` 修复 · `docs:` 文档 · `refactor:` 重构
- **不堆积**：不等到多个功能完成才一次性提交

### 自动化验证
- **每次完成功能**：运行 `python _test/run_verify.py` 确认无语法错误
- **新脚本**：必须在 `_test/` 下添加对应的验证用例
- **hooks 目录**：`.claude/hooks/` 存放项目级钩子，详见目录内 README

### MCP 工具（settings.json 配置）
本项目配置了以下 MCP 服务器，提供额外能力：

| MCP 服务器 | 用途 |
|---|---|
| `filesystem` | 项目文件系统操作（读写各子目录文件） |
| `sequential-thinking` | 结构化思维推理（教学计划、方案设计） |

如需新增 MCP（如联网搜索），需在 `.claude/settings.json` 的 `mcpServers` 中添加配置。
