# 生成 Chris 每日计划

> 调用方式：`/chris-daily-plan` 或用户在 VSteach 项目中说"生成每日计划"

## 1. 前置工作

1. 进入工作目录 `D:\@VSwork\VSteach\`
2. 读取以下上下文：
   - `_CHRIS/CLAUDE.md` — 培养目标、教学规则、科目策略
   - `_CHRIS/Chris的每日记录.md` — 最近一周完成情况，据此规划次日
   - 上一日 docx（如 `Chris每日计划YYYYMMDD.docx`）— 了解已排结构
   - `_邦共用/豆包智能体设置.md` — 豆包智能体名称和用法（发音训练官/练字考试官/古文考试官等）

## 2. 计划生成规则

### 2.1 基础原则
- **暑假节奏**：晨练 → 上午学习 → 中午篮球 → 下午学习 → 晚上活动
- **Chris 特性**：7岁好动、需要弹性、注意力在感兴趣的内容上集中
- **单次学习 ≤1.5h**，穿插休息（10min）和运动
- **休息用 ☕ 图标/早餐用 🍳/午餐用 🍱/晚餐用 🍽️**

### 2.2 练字规则（固定）
- 分 **2 次 × 30min**（共 1h）
- 内容 = **前一天背诵的古文**
- ≥ 2 页（米字格/田字格）
- 注意要点：
  - 中线对齐（Chris 字向左歪）
  - 大小一致
  - 笔画力度（偏轻，需加重）
- 写完拍照发「Chris-练字-考试官」评分
- 字帖用 `_邦共用/邦共用_字帖生成器.html` 生成

### 2.3 语音训练规则（固定）
- **不**单独列 RE（Reading Explorer）阅读项
- 用 **Chris-英语发音-训练官** 智能体训练 **RE 课文读音**
- 材料 = RE2 当前单元（按进度推进）
- 注意要点：
  - 一句一句来，读时绝不打断
  - 一次只纠 1 个最突出问题
  - 流程：单词攻克 → 整句修正 → 3~5 句连读练段落节奏
  - 全程美音（General American）标准

### 2.4 记忆曲线（复习栏）
- **只保留古文**，不含新概念和 Reading Explorer
- 格式：`📜 《篇名》 · 间隔天数（次日/2天/4天）`

### 2.5 学而思时间
- 安排在 **13:00-15:00（2h）**，其他任务顺延

### 2.6 编号规则
- 主任务编号：`D{天}C{序号}`（如 D4C1、D4C2…）
- 早餐/休息/午餐/晚餐等不编号

## 4. Word 排版规范（一页紧凑）

### 4.1 页面设置
- 上边距 1.2cm，下边距 1.0cm，左右 1.5cm
- A4 纸

### 4.2 标题区
- `C h r i s` — Georgia 22pt 金色 #B78B4A 加粗
- `每日计划` — 微软雅黑 12pt 深蓝 #2D3E50 加粗
- 日期行：`──── 2026年7月X日 周X · 第X天 ────` — 9pt 深蓝 居中
- 分隔线：`━`×49 — 6pt 深蓝 居中

### 4.3 任务表
- 表格边框：上下单线 #E8EAEB 0.5pt
- 单元格背景：白色 #FFFFFF
- 行高：13pt（紧凑）
- 列宽：编号 1.3cm / 图标 0.8cm / 时间 2.5cm / 时长 1.2cm / 内容 11.7cm
- 表格总宽：~17.5cm
- 底部三栏（复习/提问/完成）：各 5.9cm，8pt 标题金色加粗，内容 7.5pt 深蓝

### 4.4 备注区
- 标题金色加粗 8pt：`✍ 练字要点` / `🗣 语音训练要点`
- 内容 7.5pt 深蓝，每条以 `· ` 开头

### 4.5 颜色值
- 金色 `#B78B4A`、浅金圆点 `#C9B99A`、深蓝 `#2D3E50`、边框 `#E8EAEB`

## 6. 字帖 PDF 自动生成

每次计划完成后，**自动生成当天背诵古文**的字帖 PDF（辅助当日学习），而非练字内容（练字写前一天古文，PDF 辅助当天学习）。

### 6.1 自动生成逻辑
- 计划表中当天「古文」栏的篇目 → 查库 → 生成 PDF
- 例如 7/9 当天学《爱莲说》→ 自动生成 `字帖_爱莲说.pdf`

### 6.2 手动单独生成
如需某篇古文的手写字帖（例如练字用的前一天古文），单独调用：
```bash
cd _CHRIS/__temp && python gen_calligraphy.py 古文名
```
例如：`python gen_calligraphy.py 陋室铭` → 生成 `字帖_陋室铭.pdf`

### 6.3 内置古文库
脚本 `_CHRIS/__temp/gen_calligraphy.py` 内置了 20+ 篇常见古文全文（小升初范围+Chris 已学+常见篇目），直接按名称调用。

### 6.4 加入新古文
如果当前篇目不在库中，需手动输入全文。用以下格式将其加入库中：
```python
"篇名": "全文内容",
```
插入到 `GUWEN_DB` 字典中第 11 行附近。

### 6.5 输出文件
| 文件 | 路径 | 格式 |
|---|---|---|
| 字帖 PDF | `_CHRIS/字帖_篇名.pdf` | A4，每字描红×2 + 临写×2 |
| 临时 HTML | `_CHRIS/__temp/字帖_篇名.html` | 生成后自动删除（转 PDF 失败时保留供手动打印） |

### 6.6 生成步骤
1. 从每日计划提取**当天背诵的古文**篇名
2. 查 `gen_calligraphy.py` 库中是否有全文
3. 有 → 直接调用生成 PDF
4. 无 → 获取全文后临时调用，并更新古文库

### 6.7 列出所有古文
```bash
python gen_calligraphy.py --list
```

## 7. 空余时间的建议

按优先级推荐填充：

| 时间块大小 | 推荐活动 | 理由 |
|---:|---|---|
| <15min | ☕ 休息 | 7岁专注力需要频繁小休 |
| 15-30min | 💻 编程 Scratch / 🧩 数独 / 📖 英语自由阅读 | 兴趣导向（code.org 已通关，可过渡到 Scratch）；数独 Chris 喜欢；英语自由阅读保持蓝思 |
| 30-45min | 💻 Scratch 小项目 / 🎲 桌游 / 📘 英语原版阅读 | 可安排一个小任务（如 Scratch 角色动画），不宜超过 45min |
| 45min+ | 建议拆分为 30min 活动 + 15min 自由 | 7岁不宜连续超过 45min 久坐 |

**每天必须保留**：
- 纯自由/休息时间 ≥ 30min（不安排任何学习任务）
- 运动时间 ≥ 2h（分散在晨练+中午+晚上）

## 8. 生成步骤（逐日执行）

```
① 读学习记录 → 确认昨日完成和评分
② 基于昨日复盘确定次日各科进度
③ 用 gen_day9_v2.py 模板生成 Word（修改：天数/日期/周几/内容/复习项）
④ 追加当天文字版到 Chris的每日计划.md（不新建文件）
⑤ 自动生成当天古文字帖 PDF（python gen_calligraphy.py 篇名）
⑥ 验证 Word 内容正确（python dump 检查）
⑦ 清理临时文件
```
