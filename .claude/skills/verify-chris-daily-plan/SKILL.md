# 验证每日计划生成（闭环约束）

> 调用方式：用户说"生成每日计划"时自动触发，或说"验证计划"、"检查一下"
> 完整流程：生成计划 → 验证 → 有错修正 → 重新验证 → 全部通过 → 交付
> 注意：所有规则均有记录文件，切勿凭记忆推算，必须读取源文件确认

## 验证清单

### 1. 📜 古文进度验证
- 读取每日记录中最近完成的古文名称
- 读取暑假计划编号列表，找到该篇目的编号
- 确认：计划中新篇 = 编号列表中的下一篇（按编号递推）
- 确认：**字帖PDF = 练字内容 = 已背诵[-1]**（三值一致，来自递推规则）
- 确认：练字内容 ≠ 新篇（新篇只读不写，明天才进入练字）
- 确认：字帖PDF 已生成且文件名与练字内容匹配

### 2. 📘 新概念3进度验证
- 读取每日记录中最新完成的 NC3 Lesson 编号
- 确认：计划中 NC3 课文 = Lesson N+1

### 3. 🗣️ Reading Explorer 2进度验证
- 读取每日记录中最新完成的 RE 单元
- 确认：RE2 单元结构为 U{N}A+B，**没有 C 部分**
- 确认：计划中 RE 单元 = 最新完成单元的下一单元

### 4. 📚 学而思进度验证
- 读取每日记录中最新完成的学而思记录
- 确认：**第几集（volume）**——当前为第9集，不经常变
- 确认：**第几讲（lesson）**——每天变动
- 递进逻辑：前半部分 → 后半部分 → 下一讲前半部分
- 格式：学而思第{集}集 第{讲}讲

### 5. ✍️ 练字验证
- 确认：练字内容 = 已背诵[-1]
- 确认：练字2次/天，每次30min

### 6. 📄 字帖文件名编号验证
- 读取暑假计划编号列表，建立 {篇名 → 编号} 映射
- 检查所有 `字帖_{编号}_{篇名}.pdf` 文件
- 确认：文件名中的编号 = 暑假计划中该篇目的正确编号
- 确认：练字字帖PDF（新篇辅助字帖同理）文件名编号正确
- 如发现编号错误，自动修正文件名

## 闭环流程

每次用户说"生成每日计划"时，按以下步骤执行：

```
┌─ Step 1: 读取记录 ───────────────────────────┐
│  读取 Chris的每日记录.md 中所有科目的最近完成情况  │
└──────────────────────────────────────────────┘
                       ↓
┌─ Step 2: 生成计划 ───────────────────────────┐
│  根据递推规则生成每日计划 Word + 字帖 PDF        │
└──────────────────────────────────────────────┘
                       ↓
┌─ Step 3: 验证 ───────────────────────────────┐
│  运行 verify_daily_plan.py                      │
│  检查所有科目进度是否正确                        │
└──────────────────────────────────────────────┘
                       ↓
              ┌── 全部通过？──┐
              │              │
              Yes            No
              │              │
              ↓              ↓
      ┌─ Step 5: ┐   ┌─ Step 4: ──────────┐
      │  交付给用户 │   │  修正错误            │
      │  输出报告   │   │  重新生成 + 重新验证  │
      └──────────┘   └────────────────────┘
```

## 关键注意事项

1. ⚠️ **所有规则必须读取源文件，凭记忆推算会产生错误（已多次证实）**
2. ⚠️ 古文列表编号以暑假计划文件中的列表为准，不是 `gen_calligraphy.py` 的字典顺序
3. ⚠️ RE2 只有 U{N}A+B，**没有 C**，不要错误递进到 C
4. ⚠️ 学而思是"第9集 第14讲"格式，不是"14级第9集"或"9级第15讲"
5. ✅ 练字PDF文件名 = `字帖_{编号}_{篇名}.pdf`，编号来自暑假计划
