# 古文背诵理解字帖生成器

> 调用方式：`/guwen-calligraphy` 或用户说"生成古文背诵理解字帖"、"仿照增广贤文字帖生成"

## 1. 字帖格式（4行结构）

每一条目占一个卡片块，从上到下 4 行：

```
 ①  原文描红                           ← 描红（26px，浅粉色 #E0B0B0，楷体）
    白话解释（简单易懂）                  ← 灰色 #666，10px
 🏠 场景举例（贴近小学生日常生活）          ← 灰色 #888，9px
 💡 大宝提示（针对该条的教育提醒）          ← 金色 #B78B4A，8.5px
```

**注意：不要重复写原文参考行**，描红行就是原文本身。

## 2. 数据格式

每一条目用 4 个字段，竖线分隔：
```
原文|白话解释|场景举例|大宝提示
```

分类依据 Markdown 的 `## 标题` 分割，一个分类一章节。

## 3. 生成步骤

### 方案 A：有现成分类文件（推荐）
把分类好的数据存入 Markdown，格式：
```markdown
## 分类名1
- 原文|解释|场景|提醒
- 原文|解释|场景|提醒

## 分类名2
...
```
然后调用：
```bash
cd _邦共用 && python gen_calligraphy_card.py "分类文件路径.md"
```

### 方案 B：作为 AI skill 自动生成
1. 读取或接收原文内容
2. 对每条内容补充：白话解释、场景举例、针对提醒
3. 按 4 字段格式组装数据
4. 写入临时 Markdown 或直接在脚本中定义
5. 调用 `gen_calligraphy_card.py` 生成 PDF
6. 输出到 `_邦共用/@背诵库/@中文/{篇目名}/`

### 方案 C：有 Word/文本参考文件
1. 读取参考文件，提取原文句子
2. 对每条补充解释/场景/提醒
3. 组装后调用通用工具生成

## 4. 输出目录

```
_邦共用/@背诵库/@中文/{篇目名}/
├── {篇目名}_原文.docx        ← 原始文本（可选）
├── {篇目名}_分类.md          ← 分类数据源（可选）
└── {篇目名}_字帖N_分类名.pdf  ← 字帖 PDF
```
