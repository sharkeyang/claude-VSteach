第 2 课 · 构建知识图谱

基本命令

/graphify .                     # 当前目录
/graphify D:\path               # 指定路径
/graphify https://github.com/.. # 克隆 GitHub 仓库再建图

建图流程

  1. 检测文件 — 自动分类代码/文档/PDF/图片/视频
  2. AST 提取 — 代码文件走语法树（免费，无需 LLM）
  3. 语义提取 — 文档/PDF/图片由 LLM 子 agent 并行提取概念和关系
  4. 建图+聚类 — 合并结果，构建网络图，自动 Louvain 社区检测
  5. 健康检查 — 诊断 dangling edges、缺失节点
  6. 社区命名 — 给每个社区起可读名称
  7. 导出 — graph.json + GRAPH_REPORT.md + graph.html

输出产物

文件说明
graph.json原始图谱数据（可被 NetworkX 读取）
GRAPH_REPORT.md审计报告：God Nodes、社区、隐藏关系、知识缺口
graph.html交互式可视化图（浏览器打开）

实际案例：_CHRIS 目录

Corpus: 63 files · ~40K words
  code:    6 files
  docs:    43 files
  papers:  11 files (PDF 字帖)
  images:  1 file

结果：173 节点 · 252 边 · 13 社区
  AST 提取: 38 节点, 69 边（代码结构）
  语义提取: 135 节点, 204 边（文档概念）

处理大项目

如果文件超过 500 个或字数超过 200 万，graphify 会提示你缩小子目录。建议先对子目录跑，摸清效果再扩到整个项目。

⚠️ 注意编码问题： 中文 Windows 上文件路径含中文时，输出重定向可能产生 GBK 编码问题。建议在命令中指定 UTF-8 编码（如 python -X utf8）。
