第 3 课 · 查询与探索图谱

自然语言查询

图建好后，直接用日常语言提问：

/graphify query "Chris 的弱项和对应的训练方案是什么？"
/graphify query "小升初三公的证书要求有哪些？"
/graphify query "写字短板从识别到解决方案的完整链路"

查询模式

  • BFS（默认） — 广度优先，获取广泛上下文，适合"有哪些相关"
  • DFS（--dfs） — 深度优先，追踪特定路径，适合"完整链路"
  • --budget N — 限制回答长度上限（token 数）
/graphify query "AMC8 和 AMC10 的关系" --dfs
/graphify query "有哪些训练工具" --budget 500

路径查询

找两个概念之间的最短路径：

/graphify path "写字短板" "衡水体"
/graphify path "三公" "AMC8"

节点解释

解释某个节点是什么：

/graphify explain "学而思秘籍9-10级"
/graphify explain "神秘考SMK"

快速查询（已有图谱时）

图谱已经建好后，直接输入 /graphify query "..." 即可，不用重新建图。graphify 会自动检测 graphify-out/graph.json 是否存在，跳过建图步骤直接回答。

真实案例：3 次查询

  1. "Chris 的弱项" → 找到 写字短板（mitigated by 衡水体方案）+ 游泳（有待加强）+ 看图说话综合阶段
  2. "写字短板完整链路" → 识别 → 衡水体方案 → 字帖生成器 → 练字考试官智能体 → 每日计划执行，4 层链路完整 trace
  3. "小升初三公路径" → 三公目标 → 上实/上外/浦外 → 证书要求(AMC8/小托福/思维100/古诗文大赛) → 学术准备(学而思/新概念/RE/古文)
💡 查询技巧： 图谱回答问题只会引用 graph.json 中已有的边和节点。如果答案不完整，说明某些文件没被提取到，可以重建或手动补充。
