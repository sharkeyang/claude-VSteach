第 4 课 · 增量更新与高级选项

--update：增量更新

改了几个文件后，不用全量重跑：

/graphify D:\project --update

graphify 会对比文件修改时间，只重新提取新增/修改的文件，保留已有的缓存结果。适合日常迭代。

--mode deep：深度提取

/graphify D:\project --mode deep

更激进地添加 INFERRED 边——间接依赖、共享假设、潜在耦合。适合需要彻底理解架构关系时使用。

--cluster-only：只重新聚类

/graphify D:\project --cluster-only

不重新提取，只对已有的 graph.json 重新运行社区检测算法。适合调整社区划分时使用。

--directed：有向图

/graphify D:\project --directed

构建有向图，保留边的方向（source→target）。默认是无向图。有向图能看出"谁调用了谁"，但会丢失一些社区检测效果。

--watch：自动监听

/graphify D:\project --watch

监听文件变化，自动重建（无需 LLM）。适合开发过程中保持图谱实时更新。

--no-viz：跳过可视化

/graphify D:\project --no-viz

只生成 graph.json 和 GRAPH_REPORT.md，不生成 HTML 交互图。适合纯 CI 或批量处理场景。

GitHub 仓库

/graphify https://github.com/owner/repo
/graphify https://github.com/owner/repo --branch develop

多仓库合并

/graphify repo1_url repo2_url

克隆多个仓库，各自建图，然后合并成一个跨仓库图谱。

选项速查

选项作用
--update增量更新，只处理变更文件
--mode deep深度提取，添加更多推断边
--cluster-only只重新聚类
--directed构建有向图
--watch监听文件变化自动重建
--no-viz跳过 HTML 可视化
--svg导出 SVG 图
--graphml导出 GraphML 格式
⚠️ 缩容保护： 如果新图节点数比旧图少，graphify 会拒绝覆写以防止意外丢失数据。如需强制重建，加 --force。
