第3课：5 分钟自查清单

每次收到 Claude 的文档，按这个流程检查

第一步：全局浏览（30秒）

在 VS Code 中按 Ctrl+Shift+V 预览，用眼睛扫一遍：

  • 表格有没有歪的、缺列的？
  • 有没有大段灰色背景？（代码块没闭合）
  • 列表层级对不对？（子项缩进一致吗）
  • 标题层级有没有跳？（# → ### 跳过 ##）
  • 有没有乱码？（中文显示为 ?? 或 æ®�）

第二步：检查表格（1分钟）

在 bash 中执行：

# 检查每行列数是否一致 awk -F'|' '{print NF-1}' 文件名.md | sort | uniq -c

正常输出应该是同一数字。出现多个不同的数字 → 列数不一致。

第三步：检查标签闭合（1分钟）

# 统计 small 标签 grep -c '<small>' 文件名.md grep -c '</small>' 文件名.md # 两个数字必须相等

第四步：检查代码块（1分钟）

# 统计 ``` 数量 grep -c '```' 文件名.md # 必须是偶数！

第五步：检查特殊字符（1分钟）

在 VS Code 中：

  • 搜索零宽空格字符（U+200B，肉眼不可见）
  • 搜索英文双引号是否混入中文文本
  • 检查 &nbsp; 是否被误用

第六步：最终预览（30秒）

按 Ctrl+Shift+V 再完整看一遍渲染效果，确认修复生效。

💡 建议：将自查清单打印出来贴在显示器旁边，每次生成文档后过一遍。
