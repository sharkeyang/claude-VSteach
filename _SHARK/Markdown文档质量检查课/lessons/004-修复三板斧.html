第4课：常见修复三板斧

三招解决 90% 的格式问题

斧1表格列对齐

问题：列数不匹配或分隔行格式不对。

修复前：
| 原文 | 解释 | 提醒 | ← 3 列
|:---|:---|:---|
| 好言一句 | 好话让人暖 | ← ❌ 少一列
修复后：
| **原文** | **解释** | **提醒** |
|:---|:---|:---|
| **好言一句** | 好话让人暖 | 你随口一句 | ← ✅ 3 列

方法：找到出问题的行，数一下有几个 |，和表头保持一致。

斧2补齐代码块反引号

问题：代码块没闭合，后面内容变灰色。

```python print("hello") ← 少了结束的 ```
修复：在代码块末尾加上 ```

检查：搜索 ```，出现次数必须是偶数。

斧3闭合 HTML 标签

问题：<small> 或 <b> 没有闭合，导致后续文字全部变小或变粗。

<small>这句话会变小 ← ❌ 没闭合
✅ <small>这句话会变小</small>

检查：

# 统计数量 grep -c '<small>' 文件名.md grep -c '</small>' 文件名.md # 两个数字必须相等

补充：Python 引号冲突修复

当场景举例中包含 " 时，Python 字符串会报错：

scene = "同学说"你好"他来了" ← ❌ 引号冲突

三种修复方法：

方法1：用单引号包裹
scene = '同学说"你好"他来了' ✅
方法2：转义
scene = "同学说\"你好\"他来了" ✅
方法3：替换为中文括号（推荐）
scene = "同学说「你好」他来了" ✅
