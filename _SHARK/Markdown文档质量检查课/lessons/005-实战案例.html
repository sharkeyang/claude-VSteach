第5课：实战案例

真实发生过的文档格式问题复盘

案例1 621 条空解释

项目：增广贤文分类文档

症状：文档有 758 条，但 621 条的解释是空的（<small></small>）。

# 检查空标签 grep -c '<small></small>' 增广贤文_分类.md 输出：621

根因：生成脚本的关键词匹配没覆盖到这些条目，走了默认值兜底，但默认值是空的。

修复：加入 300+ 条关键词匹配规则，确保每条都有基于原文内容的独特解释。

💡 教训：不要相信"默认值"，每个条目都应该有具体内容。

案例2 Python 引号报错

项目：生成字帖脚本

症状：

SyntaxError: invalid syntax. Perhaps you forgot a comma? (gen_zgxw_calligraphy.py, line 17)

根因：场景举例中包含 "老师骂你了" 等 ASCII 双引号，与 Python 字符串界定的双引号冲突。

"同学跑来跟你说"老师骂你了"他也会说" ← ❌
修复后："同学跑来跟他说「老师骂你了」他也会说" ← ✅
💡 教训：中文文本中混入英文引号是最常见的 Python 脚本错误。

案例3 表格多出一行

项目：日常 Markdown 文档

症状：表格最后多了一行空白行或多了一列。

| 原文 | 解释 |
|:---|:---|
| 好言一句 | 好话 |
|          | ← 这一行有空格，被渲染成了空行！

修复：删除表格和下一个段落之间的多余空行或空格。

案例4 中文路径找不到文件

项目：Python 脚本访问文件

症状：

FileNotFoundError: [Errno 2] No such file or directory: 'D:\\@VSwork\\VSteach\\待整理目录\\@【杨邦】背诵文件\\...'

根因：Windows 的 bash 终端无法正确编码中文路径中的特殊字符（【】）。

修复：

方法1：用 Python 的 os.listdir() 确认实际路径
方法2：用 glob.glob() 通配符匹配
方法3：将文件移到无特殊字符的路径下
💡 教训：中文路径在 bash 中经常出问题，尽量使用全英文路径。
