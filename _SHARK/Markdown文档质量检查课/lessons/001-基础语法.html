第1课：Markdown 完整语法

覆盖昭明路线图.md 中用到的全部语法 · 每个都带正误示例

一、YAML Frontmatter（元数据区）

文档开头用 --- 包裹的元数据块，定义文档属性，不会被渲染出来。

---
name: full-roadmap
description: 全流程量化交易系统
metadata:
  type: project
---
❌ --- 前后没有空行，导致 frontmatter 不生效，内容被当作文本渲染。
💡 检查方法：预览时如果看到文档顶部出现了 name: xxx 这样的文字，说明 frontmatter 没解析成功。

二、标题层级

# 一级标题 ← 文档标题，只用一次
## 二级标题 ← 大章节
### 三级标题 ← 小章节
#### 四级标题 ← 子章节
✅ 昭明路线图的实际用法：
# 昭明路线图 — 全流程量化交易系统
## 一、使命与目标
### 1.1 使命
❌ #标题 ← # 后没空格
❌ 从 ## 跳到 #### ← 跳级了，目录会乱

三、YAML 与 HTML 混用（常见于昭明文档）

昭明文档中经常在 Markdown 内嵌入 HTML 标签来控制样式。这是 Markdown 的扩展功能，但不是所有渲染器都支持。

3.1 嵌入式 CSS 样式

<style>
body { font-size: 10pt; line-height: 1.5; }
table { font-size: 9pt; }
code { font-size: 9pt; }
</style>
✅ 实际效果：在 VS Code 预览中，表格和代码字体变为 9pt，正文行距 1.5。
❌ 如果忘记写 </style>，后面所有内容都会被当 CSS 解析，页面空白！

3.2 HTML 表格（比 Markdown 表格更灵活）

昭明路线图大量使用 HTML 表格，内部可以嵌套 Markdown 表格：

<table border="1" cellpadding="8" >
<tr><td>

| 列1 | 列2 | ← 内部是 Markdown 表格
|:---|:---|
| 数据 | 数据 |

</td></tr>
</table>
❌ HTML 标签不闭合（如 <tr> 没有 </tr>），会导致整个页面渲染错乱。

3.3 内联颜色样式

<span style="color:red"> > **警示一：** 此划分方式已确定 </span>

效果：红色文字，常用于警示

❌ <span style="color:red">文字 ← 没有 </span>，后面所有文字都变红！

3.4 不换行标签 <nobr>

用于表格中防止文字换行：

<nobr>✅ 已完成</nobr>

四、列表

4.1 无序列表（多级嵌套）

- 一级项目
  - 二级项目（缩进2空格）
    - 三级项目（缩进4空格）
- 回到一级
✅ 昭明路线图的实际嵌套：
- [一、使命与目标](#一使命与目标) ← 一级
  - [1.1 使命](#11-使命) ← 二级
    - [1.1.1 子项](#111-子项) ← 三级
❌ - 一级
 - 二级 ← 只缩进1空格，渲染异常
   - 二级 ← 缩进3空格，和2空格混用

4.2 有序列表

1. 第一步
2. 第二步
3. 第三步
❌ 1. 第一步 / 1. 第二步 / 1. 第三步 ← 编号全用1，虽然渲染正常但语义错误

4.3 任务列表（Checklist）

- [ ] 待办事项
- [x] 已完成事项
- [ ] 另一个待办

效果：☐ 待办事项 · ☑ 已完成事项

✅ 昭明路线图的实际用法：
| 里程碑3 | 三层策略验证完成 | ⏳ 待启动 | ← 用表格 + 状态图标
❌ - [x]已完成 ← [x] 后面没空格，checklist 不生效

4.4 列表内嵌链接（目录常用）

- [一、使命与目标](#一使命与目标)
  - [1.1 使命](#11-使命)
  - [1.2 三大目标](#12-三大目标)
    - [1.2.1 子目标](#121-子目标)
💡 锚点链接的规则：中文去掉标点，空格用 - 代替，如 #一使命与目标。

五、表格（Markdown 标准）

5.1 基本表格

| 列1 | 列2 | 列3 |
|:---|:---|:---|
| 数据1 | 数据2 | 数据3 |

对齐方式：:--- 左对齐 · :---: 居中 · ---: 右对齐

5.2 带对齐的表格（昭明路线图常用）

| # | 目标 | 描述 | <nobr>状态</nobr> |
|:---:|------|------|:----:|
| 🎯 | 完全脱离人为决策 | 选股/择时系统化 | ✅ 已完成 |

效果：第一列居中，第二、三列左对齐，第四列居中。

5.3 表格常见错误

❌ 列数不匹配：
| 原文 | 解释 | 提醒 | ← 3列
| 好言一句 | 好话让人暖 | ← ❌ 只有2列

❌ 分隔行格式不对：
| 原文 | 解释 |
--- | --- | --- ← 没有 | 开头，不渲染

六、引用（Blockquote）

6.1 单行引用

> 在已有操作规则的基础上，完善操盘链条。

6.2 多行引用

> 第一行
> 第二行
> 第三行

6.3 嵌套引用

> 外层引用
> > 内层引用（嵌套）
> 回到外层

6.4 引用内嵌加粗

> **注一：** 下破 DJC 应将浮仓退出；当处于 DJC 之下且 DJE 之上，只可能是 `_多长` 基仓。
❌ > 引用内容
第二行没有 > ← 变成普通段落了，不属于引用

七、代码与代码块

7.1 行内代码

使用 `_多长` 策略，依赖 `WXCD` 指标

效果：使用 _多长 策略，依赖 WXCD 指标。

DJE之上 (DXZE>0)
│

│ └── ZA≤0 → 持被动

```python
def calc_strategy():
    return score
```
❌ ```python 没有对应的 ``` 结束 → 后面所有内容都变灰色！

八、内联格式

语法写法效果常见错误
粗体**文字**文字**文字*（少一个*）
斜体*文字* 或 _文字_文字*文字（未闭合）
行内代码`代码`代码` 不匹配
删除线~~文字~~文字~~ 未成对
链接[text](url)textURL 中有空格或中文
图片![alt](img.png)—忘记 !

九、特殊字符与转义

字符用途写法常见错误
&nbsp;空格&nbsp;在 Markdown 中多空格不生效，必须用 &nbsp;
&gt;>&gt;在代码示例中直接写 > 会被当引用
&lt;<&lt;同上
&amp;&&amp;直接写 & 可能被当 HTML 实体
&#x200B;零宽空格不可见字符从网页复制时带入，破坏表格排版

十、水平分割线

---

三个或以上的 - 就是分割线。如果上面一行也是 ---，会被识别为 YAML frontmatter。

❌ --- 在文档开头被当 YAML，在中间才是分割线。如果中间不小心多写了 --- 且上下有空行，就是分割线。

🔴 避免  🟢 优先  🟡 谨慎
💡 Emoji 在 Markdown 中直接粘贴即可，不需要额外语法。但不同终端的 Emoji 渲染效果不同，VS Code 和浏览器可能显示不一样。

十三、昭明文档中的特殊模式

13.1 表格内嵌 HTML 表格

昭明文档经常用 HTML 表格包裹 Markdown 内容：

<table border="1" cellpadding="8" >
<tr><td>

**疑惑①：DJA之上出现跌排，是否应归类为卖出？**

直觉：DJA之上没有止损点...

</td></tr>
</table>

检查重点：HTML 标签必须成对出现。一个 <tr> 对应一个 </tr>。

13.2 彩色警示

<span style="color:red">
> **警示一：** 此划分方式已确定，不需要再改变。
</span>

注意：引用 > 在 <span> 内部，实际渲染时如果 span 没闭合，整个页面变红。
