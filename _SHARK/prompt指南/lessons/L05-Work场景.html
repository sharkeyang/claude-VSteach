📜 起源：什么时候该把 AI 当成"项目经理"？

👤 助理 vs 👔 项目经理

助理（Chat）：
"帮我查一下复利是什么。" → 30 秒给你答案。
"帮我写封邮件婉拒邀请。" → 2 分钟给你草稿。

项目经理（Work）：
"用这沓季度报告做一份管理层简报加六页幻灯片，受众是高管，每个数字标出来源，简报和幻灯片口径要一致。"
→ 他会先规划、收集信息、生成文件、交叉检查、终检——最终给你能直接用的成品。

什么时候用 Work？OpenAI 官方说：
需要调多个来源或工具、包含一串步骤、要做修改、或者要产出较大交付物的任务。

Work 会消耗更多 credits，但只要它省了时间、提了质量、帮你做了重要决定，就是值得的。

📖 Work Prompt 写法

Work Prompt 结构：

  1. 描述你要的结果 — 不是步骤，是最终产出
  2. 给出源材料 — 附件、链接、数据
  3. 点名受众 — 谁会用这个产出？
  4. 说明你怎么审 — 终检要求
  5. 让 AI 先规划再执行 — 做法重要时先要计划
  6. 收尾前检查 — 自查一遍再交

高效使用 Work 的要点

  • 只带相关的来源，合适的话限定日期范围
  • 定好受众、输出格式和期望长度
  • 把必须做的活，和可选的优化/润色分开
  • 做法本身重要时，先要一份计划
  • AI 发送/发布/修改别人依赖的信息前，必须经你批准
  • 它开始干你不再需要的活时，收窄任务或直接叫停
  • 先审第一版结果，打磨指令，跑通了就复用这套流程

场景一：把源材料变成成品文件

Prompt：

用附件里的季度报告，做一份管理层简报和一份六页的演示文稿。受众是高管团队。开头先给他们需要拍板的三个决策，报告里的事实和你的分析要区分开，每个数字都注明来源文件，收尾前检查简报和幻灯片口径一致。

这条涵盖：🎯目标（简报+幻灯片）→ 📚上下文（季度报告）→ 📄输出（受众、格式、注明来源）→ 🚧边界（事实vs分析分开）→ ✅终检（口径一致）。

场景二：为一个决策做调研

Prompt：

为一家 50 人的公司调研三个客服平台。用最新的资料比较定价、安全性、集成能力和迁移成本。产出一份推荐备忘录，附上链接、假设，以及签合同前我们该回答的问题。

场景三：协调一次发布

Prompt：

根据附件里的产品简报做一份发布计划。包含时间线、负责人、依赖项、风险、公告草稿、客户 FAQ 和发布日检查清单。产出最终文件之前，把还没定的决策标出来。

重复性工作的 Work 流程

最佳实践：

  1. 先在普通任务里把 prompt 打磨好
  2. 输出稳定之后，从那个任务里设置定时执行
  3. 如果每次定时运行都该新开一个任务，就单独建一个定时任务
