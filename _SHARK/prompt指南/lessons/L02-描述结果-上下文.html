L2 · 描述结果 + 补充上下文

Prompt 指南 | 基础篇 · 让你和 AI 对齐预期的关键两步

🤔 你选：

很多人写 Prompt 上来就说"帮我做 X"，但 AI 往往猜不到你要的具体样子。
关键在于两句：你要什么结果 + 我还带了什么信息。

🔍 重头发现
从"点菜"理解为什么结果描述 + 上下文如此关键
📖 直接学知识
直接学"从结果说起"和"补充上下文"的技巧

📜 起源：为什么 AI 猜不到你要什么？

🍜 想象你在面馆点餐

"老板，来碗面。"
老板：什么面？汤面还是拌面？粗面还是细面？要不要辣？

"来碗兰州牛肉面，二细，多一点香菜，不要辣。"
老板：好嘞！

写 Prompt 跟点餐一模一样——
你说得越模糊，AI 猜偏的概率就越大。

但点餐时你不会事无巨细："第一步，烧水。第二步，下面。第三步，捞起来……"
你说的是结果（"二细兰州牛肉面"），不是步骤（"第一步烧水"）。

这就是 OpenAI 官方指南的第一条原则：
「从结果说起，不要从一长串步骤说起。」

那上下文呢？就像你告诉老板"我血糖高，少放糖"——这个信息会改变他做面的方式。

📖 核心技巧

一、从结果说起

先说什么，再说怎么做。 只有当过程本身重要时，才去描述过程。其他时候，给 ChatGPT 留出空间，让它自己去搜索、比对、调整。

❌ 面向步骤： 打开上个月的销售数据，找出增长最快的产品，核对库存，排个序，然后放到表格里……

✅ 面向结果： 做一张上个月各产品销量增长率的对照表，按增长率降序排列。把库存不足 100 的产品标红。

二、补充有用的上下文

只加真正相关的来源，并且说明每个来源里该取什么。

不加上下文： 写一份项目周报。

加上下文： 用附件里的本周工时数据和 Jira 看板截图，写一份项目周报。重点关注里程碑延期的任务和原因。

→ 第二行不仅给了来源，还说明了"从中取什么"（延期任务和原因）。

三、附件的正确用法

附件类型适合场景注意
文档 / PDF总结、比对、转换内容说明要从中取什么
图片 / 截图依赖视觉信息的设计、界面还原指出图里关键区域，不要只丢图过去
网页搜索答案依赖最新信息需要核对结果时，让 AI 给出来源
⚠️ 常见误区： 把几十个不相关的文件全部丢给 AI，以为它自己会挑。实际上，太多无关上下文会稀释注意力。只加真正相关的。

四、谁是受众

受众是决定输出风格的关键上下文。同一个内容，给不同的人看写法完全不同。

给技术团队： 把这些会议记录整理成一份技术决策文档，重点写架构变更和 API 调整。
给总监： 做成一页纸的摘要，开会前扫一眼就行。决策和下一步放最前面。
给客户： 写一封友好的进展通知邮件，避免技术术语。
