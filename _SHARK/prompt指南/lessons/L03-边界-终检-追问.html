📜 起源：为什么 AI 需要"防护网"？

👨‍🍳 想象你雇了一个厨师

你说"做一桌菜"，他欣然开干。结果：
— 他用了冰箱里过期两天的牛奶
— 他不小心放了三勺盐
— 他做好了直接端出去给客人吃了（你还没尝过）

这不是厨师不努力，是你没有给他边界（"别用过期的食材"）、终检（"上桌前我先尝一口"）。

但第一道菜咸了，你会直接解雇他吗？不会。你会说：
"这道菜太咸了，下一道少放一半盐。"
这就是追问——不推翻重来，只指正方向。

OpenAI 官方指南的智慧就在于此：
第一条 prompt 不需要完美。边界防大错，终检保质量，追问磨细节。

📖 核心技巧

一、设定边界

边界是那么几条指令，用来防止 ChatGPT 制造额外工作，或做出你没打算让它做的动作。

什么时候加边界？

  • 改错一个细节结果就没法用的时候 → "已批准的日期和预算数字保持不动"
  • 影响到别人之前你想先过目的时候 → "把消息准备成草稿，不要发出去"

常见的边界写法：

  • 「只用我提供的来源。信息缺了就标出来，不要猜。」
  • 「建议控制在说好的预算之内。」
  • 「不要改任何代码，只输出分析结果。」
  • 「不要发送或发布任何东西。」
⚠️ 边界要少而精： 盯住最要紧的一两条就好。不需要控制 AI 走的每一步，否则反而限制了它的能力。

二、终检

重要的工作，让 ChatGPT 收尾时做一次最终检查。

终检示例：

  • 「确认每个行动项都有负责人和截止日期。」
  • 「标记你没法核实的信息。」
  • 「检查所有链接是否有效。」
  • 「审一遍数据和结论是否一致。」

终检之后，你自己再过一遍，再去用、再去分享。AI 的终检是辅助，不是替代。

三、用追问改进结果

第一条 prompt 不需要完美。看完结果，把你想要的具体改动说出来。

追问示例：

  • 「开头写得更直接一点，证据保留，把建议挪到背景前面。」
  • 「再加一个备选方案。」
  • 「把详细程度降低，保持要点就行。」
  • 「这里的数据不对，换成附件里的最新版本。」

四、Steer 与 Queue（Codex 独有）

Codex 干活的时候，你不用等当前这轮跑完，就能发下一条消息：

方式说明适合场景
Steer（转向）把消息加进当前这一轮改方向、补细节、给新信息
Queue（排队）把消息留到下一轮等当前工作干完再说的追问

快捷键：

  • Codex CLI： 按 Enter = Steer，按 Tab = Queue
  • ChatGPT 桌面： 「设置 > 通用 > 追问行为」里选默认行为
