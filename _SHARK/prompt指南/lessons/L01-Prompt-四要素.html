📜 起源：Prompt 为什么需要框架？

🗣️ 想象一下：你让同事帮你写一份报告

假设你走到同事工位前，只说一句：
"帮我写一份报告。"

同事会一脸茫然——
什么报告？给谁看？多长？什么格式？哪些数据不能用？

你不得不补上：
"季度销售报告，给总监看的，一页纸，用最新的数据，预算数字不能动。"

这就是 Prompt 的本质：你给 AI 的信息越完整，AI 的产出越接近你的预期。

OpenAI 官方把有效 Prompt 拆成四个要素：目标、上下文、输出、边界。
这四个要素不是凭空发明的——它们就是"跟人交代工作时自然而然会说的那些话"。
💡 核心洞见： 你不需要每次都填满四项。短 prompt 往往就够了。任务越大越重要，越需要把几个要素写清楚。

📖 四要素框架

要素英文说明类比
🎯 目标GoalChatGPT 该做什么？"帮我写一份报告"
📚 上下文Context哪些信息或来源能帮上忙？"用最新的季度数据"
📄 输出Output你需要什么格式、多长、多细？"一页纸，给总监看"
🚧 边界Boundaries哪些东西不能动？哪些事该避开？"预算数字不能改"

🎯 目标（Goal）

最核心的要素。从结果说起，不要从一长串步骤说起。

❌ 差： 先打开会议记录，然后找出决策和下一步行动，然后整理成段落……
✅ 好： 把这些会议记录整理成一份给项目组的简短进展通报。决策和下一步行动放在最前面。

📚 上下文（Context）

把可能改变结果的信息给它。只加真正相关的来源，并且说明每个来源里该取什么。

不带上下文： 写一份项目状态通报。
带上下文： 用 Drive 里最新的项目计划，加上项目 Slack 频道里相关的决策和进展，准备一份状态通报。

📄 输出（Output）

告诉 ChatGPT 你打算怎么用这个结果。它就能选对长度、详细程度和组织方式。

指定输出： 做成一页纸的摘要，让总监开会前扫一眼就行。决策和下一步放最前面。
指定格式： 做成一张计划支出和实际支出的对照表，差异超过 10% 的都标出来。

🚧 边界（Boundaries）

防止 ChatGPT 制造额外工作，或做出你没打算让它做的动作。

防错边界： 已批准的日期和预算数字保持不动。只用我提供的来源，信息缺了就标出来，不要猜。
安全边界： 把消息准备成草稿，不要发出去。建议控制在说好的预算之内。

💡 四要素组合示例

完整的 Prompt：

为周一的管理层会议准备一份一页纸的项目状态通报。
用 Drive 里最新的项目计划，加上项目 Slack 频道里相关的决策和进展。
管理层需要拍板的决策和下一步行动放在最前面。
总结进度、风险、负责人和截止日期。
已批准的日期和预算数字保持不动。
有冲突或缺失的信息就标出来，任何东西都不要发送或发布。

→ 这条 Prompt 覆盖了🎯目标、📚上下文、📄输出、🚧边界，但没有把每一步都写死。
