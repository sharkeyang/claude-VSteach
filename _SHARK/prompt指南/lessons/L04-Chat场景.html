💬 朋友聊天 vs 同事开会

你跟朋友聊天：
"周末干嘛了？" "没干嘛。" "那下周有啥计划？"
— 话题随便跳，不需要背景介绍，没有 KPI。

你跟同事开会：
"好的，现在讨论 Q3 的预算调整。小王你先说。"
— 有目标、有背景、有输出要求。

Chat 模式对应的是"朋友聊天"——
适合快问快答、找点子、打草稿、日常决策。
不适合需要调多个来源、多步骤操作、生成成品文件的复杂任务。

OpenAI 官方建议：正式复杂任务用 Work 模式（下一课学），日常轻量级交互用 Chat。
Chat 的写法精髓：从你想要的结果说起，细节只在它会改变答案的时候再加。

📖 四大 Chat 场景

场景一：理解一个话题

Prompt：

给一个从没投资过的人解释复利是怎么回事。用一个具体的例子，你引入的金融术语都要给出定义。

要点：指明受众（从没投资过的人）和表达方式（具体例子 + 术语定义）。

场景二：起草和打磨文字

Prompt：

起草一封友好的邮件，婉拒这个邀请，理由是我届时在外出差。控制在 120 词以内，给以后的活动留个口子。

要点：语气（友好）、理由（出差）、长度（120 词）、后续（留口子）。

场景三：比较选项

Prompt：

帮一个每年出国两次的人比较这两个手机套餐。重要差异用表格列出来，然后推荐一个，说清代价在哪。

要点：用户画像（每年出国两次）、输出格式（表格 + 推荐 + 代价分析）。

场景四：做一个实用的计划

Prompt：

规划五顿工作日晚餐，每顿 30 分钟以内做完。不要花生，食材尽量跨餐复用，最后汇总成一张购物清单。

要点：约束明确（30 分钟、不要花生、跨餐复用）、产出明确（购物清单）。

Chat Prompt 模板速查

场景固定要素可选要素
理解话题受众 + 解释方式例子、术语定义
起草文字语气 + 长度理由、后续动作
比较选项用户画像 + 输出格式推荐、代价分析
做计划约束 + 产出物复用策略、时间限制
