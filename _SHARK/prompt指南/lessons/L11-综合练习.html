L11 · 综合 Prompt 练习

Prompt 指南 | 综合实战篇 · 从需求到完整 Prompt 的实战演练

📝 本课目标

把前面 10 课学到的所有技巧，应用到真实场景中。
每个案例给你需求背景，你来设计完整的 Prompt。

💡 自测方法： 每个案例先自己写 Prompt，然后看下方的参考答案。对比你的 Prompt 是否覆盖了目标、上下文、输出、边界、终检五个要素。

案例一：管理层汇报

需求背景： 你是市场部主管，下周要向管理层汇报 Q2 的营销效果。你有三份附件：

  • Q2 各渠道投放数据表（Excel）
  • Q2 社交媒体互动数据（PDF）
  • Q2 品牌知名度调研报告（PDF）

要求： 做一份一页纸的汇报摘要 + 五页幻灯片。

🔎 点击看参考答案

Work Prompt：

用附件里的三份 Q2 资料——投放数据表、社交媒体互动数据、品牌调研报告——做一份管理层汇报。

产出：

  1. 一份一页纸的汇报摘要（决策和关键发现放最前面）
  2. 五页演示文稿（概述、渠道表现、社交互动、品牌认知、建议）

要求：

  • 受众是管理层——他们要拍板决策，不要细节堆砌
  • 报告里的数据和你的分析要区分开
  • 每个关键数字注明来源文件
  • 投得最差的渠道和最好的渠道都标出来
  • 收尾前检查摘要和幻灯片数据一致
  • 任何东西都不要发送或发布，给我审完再说

案例二：修 Bug

需求背景： 用户反馈"点击下载按钮后，文件没下载，浏览器控制台报 500 错误"。你是后端开发，怀疑是下载接口的问题。

  • 项目路径：~/projects/file-service
  • 相关文件：routes/download.ts, services/fileStorage.ts
  • 你可以在本地复现
🔎 点击看参考答案

Codex Bug Prompt：

Bug：点击下载按钮后文件没有下载，浏览器控制台报 500 错误。

Repro：

  1. 启动服务：npm run dev
  2. 打开页面，找到带下载按钮的文件列表
  3. 点击任意文件的下载按钮
  4. 浏览器弹不出下载弹窗，控制台显示 500

相关文件：routes/download.ts, services/fileStorage.ts

Constraints：

  • 不要改 API 的返回格式
  • 修复保持最小化
  • 加一个回归测试防止重复出现

先在本地重现这个错误，定位根因，提出修复方案，跑测试验证。

案例三：从截图做原型

需求背景： 设计师给了你一张 Figma 截图——一个"个人资料编辑页"。

  • 截图上有：头像上传区、昵称输入框、个性签名文本区、保存按钮
  • 头像区域有 hover 时的"更换头像"提示
  • 保存时要做表单校验（昵称必填、签名最多 100 字）
🔎 点击看参考答案

Codex 原型 Prompt：

照这张截图做一个个人资料编辑页。

约束：

  • 用 react、vite 和 tailwind，TypeScript
  • 间距、字体、布局尽量贴近截图

截图里看不到但需要实现的行为：

  • 头像区域 hover 时显示"更换头像"的浮层
  • 点击头像区域触发文件选择器（只接受图片格式）
  • 保存时校验：昵称必填、个性签名最多 100 字
  • 校验不通过时在对应字段下方显示红色错误提示

产出标准：一份可直接运行的页面组件。

案例四：调研决策

需求背景： 你们团队想引入一个前端测试框架。备选方案：Vitest、Jest、Playwright（仅 E2E）。团队 6 人，主要用 React+TypeScript。

要求： 出一份推荐报告。

🔎 点击看参考答案

Work Prompt：

为一家 6 人 React+TypeScript 前端团队调研三个前端测试框架：Vitest、Jest、Playwright。

用网页搜索获取最新信息。比较维度：

  • 性能（测试速度）
  • React+TypeScript 的兼容性
  • 与现有 CI/CD 集成的难度
  • 社区活跃度和文档质量
  • 学习曲线

产出：一份两页的推荐备忘录。

  • 第一页：三个框架的对比表格 + 推荐方案 + 推荐理由
  • 第二页：如果选推荐方案，迁移步骤、风险和不确定因素

每条数据都注明来源链接。签合同前还需要回答的问题单独列出来。

✅ Prompt 自查清单

写完 Prompt 后，逐项检查：

  1. ☐ 我是否从结果说起，而不是从步骤说起？（L1-目标）
  2. ☐ 我是否给出了足够的相关上下文？（L2-上下文）
  3. ☐ 我是否指明了受众和输出格式？（L2-输出）
  4. ☐ 我是否设了关键边界防止 AI 跑偏？（L3-边界）
  5. ☐ 我是否要求了终检？（L3-终检）
  6. ☐ 这个任务适合 Chat、Work 还是 Codex？（L4-选择）
  7. ☐ 附件/图片是否只加了相关来源并说明了从中取什么？（L6-数据源）
  8. ☐ 如果是修 Bug，是否给了复现步骤、约束和验证方式？（L8-修 Bug）

📌 记住： 第一条 prompt 不需要完美。审结果，追问问，迭代到满意为止。
