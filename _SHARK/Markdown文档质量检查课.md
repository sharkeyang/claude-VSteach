# Shark 的 Markdown 文档质量检查课

> 面向：Shark（Claude 使用者）
> 目标：快速识别 Claude 生成的 Markdown 文档中的格式错误，掌握人工判断和修复的方法

---

## 第1课：Markdown 基础 — 5 种最常用语法

### 1.1 标题

```markdown
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
```

**正确示例：**
```markdown
# 《增广贤文》全文归类分析     ← 文档标题，用 1 个 #
## 🏆 第一优先：说话之道        ← 大分类，用 2 个 #
### 说话之道的核心原则           ← 小分类，用 3 个 #
```

**错误示例：**
```markdown
#标题          ← ❌ # 后面没有空格，不生效
## 第一优先    ← ✅ 正确
#### 子标题    ← ❌ 从 ## 跳到 ####，跳级了，目录会乱
## 第二优先    ← ✅ 正确
错误写法       ← ❌ 没加 #，被当成普通段落
```

**检查方法：** 在 VS Code 按 `Ctrl+Shift+V` 预览，看标题是否从大到小逐级排列。

---

### 1.2 列表

#### 无序列表

```markdown
- 项目一
- 项目二
  - 子项目（前面2个空格）
  - 子项目
- 项目三
```

**正确示例：**
```markdown
- 说话之道
  - 好言一句三冬暖（缩进 2 空格）
  - 伤身事莫做（缩进 2 空格）
- 处世智慧
  - 出门看天色（缩进 2 空格）
```

**错误示例：**
```markdown
- 说话之道
  - 好言一句三冬暖    ← ✅ 正确缩进
   - 伤身事莫做        ← ❌ 缩进了3个空格，不一致
    - 是非只为多开口   ← ❌ 缩进了4个空格，变成子子项目了
- 处世智慧
 - 出门看天色         ← ❌ 只缩进了1个空格

- 项目三              ← ❌ 前面有空行，但空行不对
```

#### 有序列表

```markdown
1. 第一步
2. 第二步
3. 第三步
```

**错误示例：**
```markdown
1. 第一步
1. 第二步        ← ❌ 编号重复，应该用 2.
1. 第三步        ← ❌ 编号重复，应该用 3.
```

**检查方法：** 预览时看缩进是否整齐，子项是否在正确层级。

### 1.3 表格（最容易出错的格式）

```markdown
| 列1 | 列2 | 列3 |
|:---|:---|:---|
| 数据1 | 数据2 | 数据3 |
| 数据4 | 数据5 | 数据6 |
```

**正确示例（3列）：**
```markdown
| **原文** | <small>一句话解释</small> | <small>🎯 大宝提醒</small> |
|:---|:---|:---|
| **好言一句三冬暖** | <small>好话让人冬天暖</small> | <small>你随口一句伤人的话</small> |
| **伤身事莫做** | <small>伤身体的事不做</small> | <small>你说话像刀子</small> |
```

**错误示例：**
```markdown
| 原文 | 解释 | 提醒 |        ← ✅ 3列
|:---|:---|:---|
| 好言一句 | 好话让人暖 | 你随口一句 |  ← ✅ 3列，正确
| 伤身事莫做 | 伤身体的事 |                    ← ❌ 只有2列，少了一列
| 是非只为多开口 | 是非话多 | 烦恼强出头 | 随口一句  ← ❌ 有4列，多了一列

| --- | --- | --- |         ← ❌ 分隔行少了前后的 |
```

**对齐方式的不同效果：**
```markdown
| 左对齐 | 居中 | 右对齐 |
|:---|:---:|---:|
| 文字 | 文字 | 123 |
| 长文长文 | 长文长文 | 4567 |
```

效果：
```
| 左对齐     |  居中   | 右对齐 |
|:---|---:|---:|
| 文字       |  文字   |    123 |
| 长文长文   | 长文长文 |   4567 |
```

**检查方法：** 预览时看表格列是否对齐。数每行 `|` 的个数是否相同（用 `Ctrl+F` 搜索 `|`）。

### 1.4 代码块

````markdown
```python
def hello():
    print("Hello, World!")
```
````

**要点：**
- 开始和结束都要用 **3 个反引号**（\`\`\`）
- 开始的反引号后面可以加语言名称（python/bash/json 等），启用语法高亮
- 代码块前后必须有空行

**正确示例：**
```markdown
前面是普通文字段落。

```python
print("这是代码块")
```

后面继续普通文字。
```

**错误示例：**
```markdown
前面没有空行
```python
print("代码块")
```
后面也没有空行
```

效果：列表或表格可能会被误吞进代码块。

**另一个常见错误——未闭合：**
````markdown
```python
print("hello")
```                      ← 忘记写结束的 ```

后面的所有内容都会变成灰色代码块，直到文件末尾！
```

**检查方法：** 预览时看到大段灰色背景 → 说明代码块没闭合。搜索 `` ``` `` 出现的次数应该是偶数。

### 1.5 引用

```markdown
> 这是单行引用

> 这是多行引用
> 可以跨多行
> 每行前面加 >
```

**正确示例：**
```markdown
> **大宝问题：** 经常口无遮拦，想什么说什么，不懂分寸。
```

**错误示例：**
```markdown
> **大宝问题：** 经常口无遮拦    ← ✅ 正确

> **大宝问题：** 经常口无遮拦
想什么说什么，不懂分寸。         ← ❌ 第二行没加 >，变成普通段落
```

**检查方法：** 引用的每一行开头都应该有 `>`。

### 1.6 其他常用语法

| 语法 | 写法 | 效果 |
|:---|:---|---|
| **粗体** | `**文字**` | **文字** |
| *斜体* | `*文字*` | *文字* |
| `行内代码` | `` `代码` `` | `代码` |
| ~~删除线~~ | `~~文字~~` | ~~文字~~ |
| [链接](url) | `[显示文字](https://)` | [点击这里](https://example.com) |
| ![图片](url) | `![替代文字](图片URL)` | — |

**粗体常见错误：**
```markdown
**文字*      ← ❌ 少了一个 *，整段后面的内容都变粗体
**文字**     ← ✅ 正确
```

## 第2课：Claude 最常犯的格式错误（TOP 12）

### 🔴 错误1：表格列数不一致 — 最常见

**表现：** 表格歪歪扭扭，一列跑到另一列去了，或整行无法显示。

```markdown
| 原文 | 解释 | 提醒 |        ← 3列
|:---|:---|:---|
| 好言一句 | 好话让人暖 | 你随口一句 |  ← ✅ 3列，正常
| 伤身事莫做 | 伤身体的事 |               ← ❌ 少一列！只有 2 个 |

| 原文 | 解释 |                      ← ❌ 这里只有2列了
|:---|:---|
| 好言一句 | 好话让人暖 | 你随口一句 |    ← 却有3列数据
```

**修复：** 确保每行 `|` 的数量相同。下面的脚本可以帮你检查：

```bash
# 在 bash 中检查每行列数
awk -F'|' '{print NF-1}' 文件名.md | sort | uniq -c
```

### 🔴 错误2：表格分隔行格式不对

**表现：** 表格完全不渲染，文字直接堆在一起。

```markdown
| 原文 | 解释 |          ← 表头
 --- | --- | ---         ← ❌ 分隔行没有 | 开头
| 数据 | 数据 |          ← 实际上和表头在同一行了
```

**正确格式：**
```markdown
| 原文 | 解释 |
|:---|:---|           ← 前面有 |，后面有 |
| 数据 | 数据 |
```

### 🔴 错误3：列表缩进不一致

**表现：** 子列表项跑到外面去了。

```markdown
- 主列表
  - 子列表（缩进 2 空格）
  - 子列表（缩进 2 空格）
 - 这行只缩进了 1 个空格 ← ❌ 渲染异常，和主列表平级了
    - 这行缩进了 4 个空格 ← ❌ 变成子子列表了
```

**正确写法：** 全部用 **2 个空格** 缩进一级，不要混用 2/3/4 空格。

### 🔴 错误4：代码块未正确闭合

**表现：** 后面所有内容都变成灰色的代码块样式。

```markdown
## 说话之道

​```python
# 这是一段代码
print("hello")
```
                                 ← ❌ 少了结束的 ```
后面所有内容都变灰色了...

连这个表格都变灰色：
| 列1 | 列2 |
```

**判断方法：** 在 VS Code 预览中看到大段灰色文字，立即往上找最近的 `\`\`\``，看是否有对应的结束符。

**修复：** 在代码块末尾加 `\`\`\``。

### 🔴 错误5：中英文引号混用

**表现：** 在 Python 脚本中导致 `SyntaxError`，或者文档中引号显示异常。

```python
# 下面的字符串包含中文引号 " 和 "（Unicode U+201C 和 U+201D）
scene = "同学跑来跟你说"老师骂你了"他也会说"  
#                                    ^^^^^^ 这里 Python 报错了！
```

**问题原因：** Claude 有时会在中文文本中使用 `"`（左双引号）和 `"`（右双引号），但这些字符在 Python 字符串中就是普通字符。真正的冲突是 ASCII 双引号 `"`（U+0022）出现在字符串内。

**判断方法：** 在 Python 中运行时报错 `SyntaxError: invalid syntax`，看行号定位。

**修复：**
```python
# 方法1：用单引号包裹
scene = '同学跑来跟他说「老师骂你了」他也会说'

# 方法2：转义
scene = "同学跑来跟他说\"老师骂你了\"他也会说"

# 方法3：替换为中文括号（推荐）
scene = "同学跑来跟他说「老师骂你了」他也会说"
```

### 🔴 错误6：`<small>` 或 `<b>` 标签未闭合

**表现：** 整个段落字体变小或变粗，渲染到预期外的位置。

```markdown
| **原文** | <small>一句话解释</small> | <small>🎯 大宝提醒 |
|:---|:---|:---|
| **好言一句** | <small>好话让人冬天暖 | <small>你随口一句 |
```

最后一行缺少 `</small>`，导致整个页面后面的文字都变小。

**检查方法：**
```bash
grep -c '<small>' 文件名.md
grep -c '</small>' 文件名.md
# 两个数字必须相等。不等就说明有未闭合的标签。
```

### 🔴 错误7：文件编码为 GBK 而非 UTF-8

**表现：** 中文显示为乱码，如 `ç»Ÿè®¡`、`` 等。

```markdown
文件内容看起来像：
 318 条
```

**原因：** 文件保存时用了 ANSI/GBK 编码，而 Markdown 需要 UTF-8。

**修复（VS Code）：**
1. 底部状态栏点击编码名称（如 "GBK" 或 "UTF-8"）
2. 选择「通过编码保存」
3. 选择 UTF-8
4. 重新保存

### 🔴 错误8：零宽空格（Zero Width Space）

**表现：** 表格看起来没问题，但有些行无法对齐。复制粘贴时发现奇怪的分隔。

**特征：** 这个字符 U+200B 是**不可见**的，肉眼看不到，但会影响渲染。

**检查方法：** 在 VS Code 中按 `Ctrl+F`，搜索 `​` 或直接粘贴一个零宽空格进去看。

**修复：** 删除即可。

### 🔴 错误9：超长行导致表格溢出

**表现：** 表格在预览时超出屏幕，需要左右滚动才能看完。

```markdown
| 原文 | 一句话解释 | 大宝提醒 |
|:---|:---|:---|
| 好言一句三冬暖，恶语伤人六月寒 | 好话让人冬天都暖，坏话让人夏天都寒。这句话告诉我们说话要讲分寸，不要随便出口伤人。 | 你随口一句伤人的话，别人可能记很久很久很久很久很久很久 |
```

**修复：** 将单元格内容控制在 30 字以内，或手动换行：

```markdown
| **好言一句三冬暖** | <small>好话让人冬天暖，坏话让人夏天寒</small> | <small>你随口一句伤人的话，别人可能记很久</small> |
```

### 🔴 错误10：标题层级跳级

**表现：** VS Code 大纲（OUTLINE）面板中出现断层。

```markdown
# 一级标题
### 三级标题     ← ❌ 跳过了二级，直接到三级
```

**判断方法：** 按 `Ctrl+Shift+P` → `Markdown: Open Preview to the Side`，看大纲面板。

**修复：** 确保 `#` 的个数逐级递增，不要跳跃。

### 🔴 错误11：`|` 被用在表格之外的地方

**表现：** 表格外的文字中出现 `|` 符号被误解析为表格。

```markdown
代码中出现了 | 符号 → 有时会被当表格
```

**修复：** 在 Markdown 普通段落中使用 `|` 时需要转义 `\|`。

### 🔴 错误12：URL 中包含空格或中文

**表现：** 链接点击无效。

```markdown
[错误链接](https://example.com/my file.pdf)
[错误链接](https://example.com/中文路径.pdf)
```

**修复：**
```markdown
[正确链接](https://example.com/my%20file.pdf)
[正确链接](https://example.com/%E4%B8%AD%E6%96%87%E8%B7%AF%E5%BE%84.pdf)
```

## 第3课：快速检查清单（5分钟自查法）

每次收到 Claude 的文档，按这个流程检查：

### 第一步：全局浏览（30秒）

在 VS Code 中按 `Ctrl+Shift+V` 预览，用眼睛扫一遍：

```
👀 看什么：
☐ 表格有没有歪的、缺列的？
☐ 有没有大段灰色背景？（代码块没闭合）
☐ 列表层级对不对？（子项缩进一致吗）
☐ 标题层级有没有跳？（# → ### 跳过 ##）
☐ 有没有乱码？（中文显示为 ?? 或 ）
```

### 第二步：检查表格（1分钟）

```bash
# 在 bash 中执行
# 统计每行列数是否一致
awk -F'|' '{print NF-1}' 文件名.md | sort | uniq -c
# 正常输出应该是：
#   1 行数  ← 分隔行（---）
#   2 行数  ← 表头行
#   3 行数  ← 数据行（3列表格就是3）
# 如果出现多个不同的数字，说明列数不一致
```

或者手动看：在预览中表格是否整齐。

### 第三步：检查标签闭合（1分钟）

```bash
grep -c '<small>' 文件名.md
grep -c '</small>' 文件名.md
# 两个数字必须相等

grep -c '<b>' 文件名.md
grep -c '</b>' 文件名.md
# 两个数字必须相等
```

### 第四步：检查代码块（1分钟）

```bash
grep -c '```' 文件名.md
# 必须是偶数！否则有未闭合的代码块
```

### 第五步：检查特殊字符（1分钟）

在 VS Code 中：
```bash
# 搜索零宽空格（粘贴下面的字符到搜索框）
# 或者搜索正则表达式 ​

# 搜索中文引号内混入的英文引号
# 搜索 "（英文双引号）

# 搜索 &nbsp; 是否被误用
```

### 第六步：最终预览（30秒）

按 `Ctrl+Shift+V` 再完整看一遍渲染效果。

## 第4课：常见修复三板斧

### 修复1：表格歪了

**问题：** 列表数不一致或分隔行格式不对。

**方法：** 找到出问题的行，数一下有几个 `|`，和表头保持一致。

```markdown
# 修复前：列数不匹配
| 原文 | 解释 | 提醒 |
|:---|:---|:---|
| 好言一句 | 好话让人暖 |   ← 少一列

# 修复后：
| **原文** | **解释** | **提醒** |
|:---|:---|:---|
| **好言一句** | 好话让人暖 | 你随口一句 |
```

### 修复2：代码块未闭合

**方法：** 找到开始位置的 `\`\`\``，在代码结束位置加上同样的 `\`\`\``。

```markdown
​```python
print("hello")
​```                          ← 补上这一行
```

### 修复3：HTML 标签未闭合

**方法：** 搜索 `<small>`，找到没有 `</small>` 对应的那一个。

```markdown
<small>这句话会变小</small>     ← ✅ 闭合了
<small>这句话也会变小           ← ❌ 没闭合
```

## 第5课：实战案例

### 案例1：增广贤文分类文档 — 621 条空解释

**症状：** 文档有 758 条，但 621 条的解释和提醒是空的（`<small></small>`）。

**定位方法：**
```bash
grep -c '<small></small>' 增广贤文_分类.md
# 输出：621
```

**根因：** 生成脚本的关键词匹配没覆盖到这些条目，走了默认值兜底，但默认值是空的。

**修复：** 加入 300+ 条关键词匹配规则，确保每条都有解释。

### 案例2：Python 脚本报错 SyntaxError

**症状：**
```
SyntaxError: invalid syntax. Perhaps you forgot a comma?
  (gen_zgxw_calligraphy.py, line 17)
```

**定位方法：** 看第 17 行：
```python
("来说是非者，便是是非人", "跑来跟你说别人坏话的人", "同学跑来跟你说"老师骂你了"他也会在别人面前说你坏话", "别信他")
```

**根因：** 场景举例中的 `"老师骂你了"` 包含 ASCII 双引号 `"`，被 Python 解释器当成了字符串结束符。

**修复：** 将中文引号替换为「」：
```python
("来说是非者，便是是非人", "跑来跟你说别人坏话的人", "同学跑来跟他说「老师骂你了」他也会在别人面前说你坏话", "别信他")
```

### 案例3：Markdown 表格多出一行

**症状：** 表格最后多了一行空白行，或者多了一列。

**示例：**
```markdown
| 原文 | 解释 |
|:---|:---|
| 好言一句 | 好话 |
| 伤身事莫做 | 伤身体 |
|            ← 这一行有空格的空行，被渲染成了空行！
```

**修复：** 删除表格和下一个段落之间的多余空行或空格。

### 案例4：中文路径导致 Python 脚本找不到文件

**症状：**
```
FileNotFoundError: [Errno 2] No such file or directory: 
  'D:\\@VSwork\\VSteach\\待整理目录\\@【杨邦】背诵文件\\...'
```

**根因：** Windows 的中文路径在 bash 中编码异常。

**修复：** 用 Python 的 `os.path.exists()` 先检查，或者用短路径。

在 VS Code 的 `settings.json` 中启用以下设置：
