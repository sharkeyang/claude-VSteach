第 3 课 · 上下文窗口与项目管理

什么是上下文窗口？

每次和 Claude 对话，聊过的内容都存在一个"临时记忆"里。这个记忆空间就叫上下文窗口。Claude 的上下文窗口非常大（可达 200K tokens），大约能容纳 2000 行代码或一整本 500 页的书。

为什么这很重要？

  • 你可以把整个项目交给 Claude，它记得你前面说过什么
  • 但对话越长，Claude 对最开头的内容的记忆会变模糊
  • 所以关键信息要放在前面，或者在对话中重复强调

CLAUDE.md — 给 Claude 的"永久记忆"

在项目根目录创建 CLAUDE.md 文件，Claude 每次进入这个项目都会自动读到它。适合放：

  • 项目简介和背景
  • 技术栈和代码规范
  • 工作流程和规则
  • 常用命令和路径
💡 CLAUDE.md 实用示例：
# 项目规则
- Python 使用 Anaconda，路径：C:\ProgramData\anaconda3\python.exe
- 数据输出到 D:\zdata\
- 不要在对话中直接运行破坏性操作
- 修改 main 函数前先口头确认方案

管理长对话的技巧

  • 新对话 vs 续旧对话 — 如果话题变了，果断开新对话，不要在同一对话里讨论完全不同的事情
  • 关键信息前置 — 把最重要的背景放在第一条消息里
  • 定期总结 — 长对话中间让 Claude 总结一下进度："到目前为止我们做了哪些决定？"
  • 存档重要输出 — 好的代码、方案随手保存到文件，不要把 Claude 当永久存储
📌 和 Harness 的关联： 这就是为什么我们在项目中配置了 CLAUDE.md + hooks + permissions——让每次对话自动加载正确上下文，不需要每次手动说明。
