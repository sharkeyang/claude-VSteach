第 9 课 · MCP — 扩展能力边界

MCP（Model Context Protocol）是 Anthropic 推出的开放协议，相当于 Claude 的"App Store"——装一个 MCP 服务器，Claude 就多一种能力。

简单说，没有 MCP 的 Claude 只能读你给它的文件和凭记忆回答；有了 MCP，Claude 可以自己打开网页、查最新文档、分步推理复杂问题。

一、核心概念

概念说明类比
MCP Server提供工具的服务进程App
MCP Tool服务器暴露的具体功能App 里的功能按钮
MCP Resource服务器可读取的数据资源App 里的文件
Stdio 传输本地启动进程，通过 stdio 通信本地 App
HTTP 传输远程服务，通过网络 API 调用网页 App
💡 怎么知道当前有哪些 MCP？ 在对话中直接问"列出你所有可用的 MCP 工具"即可。

二、添加 MCP 的 3 种方式

方式 A：claude mcp add 命令（最常用）

# 格式：claude mcp add <名称> -- <启动命令>
claude mcp add playwright -- npx @playwright/mcp@latest

方式 B：.mcp.json 文件（项目级，可提交 Git）

在项目根目录创建 .mcp.json：

{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@latest"]
    },
    "sequential-thinking": {
      "command": "npx",
      "args": ["@anthropic/mcp-sequential-thinking@latest"]
    }
  }
}

方式 C：全局 settings.json（长期使用）

编辑 ~/.claude/settings.json：

{
  "mcpServers": {
    "context7": {
      "command": "npx",
      "args": ["@anthropic/mcp-context7@latest"]
    }
  }
}

管理命令

claude mcp list           # 列出所有已配置的 MCP
claude mcp add <名称> -- <命令>  # 添加新 MCP
claude mcp remove <名称>  # 移除
claude mcp restart <名称> # 重启（卡死时有用）

三、真实案例详解

案例 1：Playwright — 浏览器自动化

让 Claude 像人一样操作浏览器——打开网页、截图、填表单。

配置：

claude mcp add playwright -- npx @playwright/mcp@latest

可用操作：

操作用途
browser_navigate导航到指定 URL
browser_snapshot获取页面结构树（比截图更适合决策）
browser_take_screenshot截图（可视化展示）
browser_click点击元素
browser_type在输入框中填写文本
browser_fill_form批量填写表单字段
browser_select_option选择下拉框选项
browser_evaluate在页面中执行 JavaScript
browser_network_requests查看网络请求

典型用法：

  • "打开这个网页，截图给我看" → 验证部署效果
  • "帮我填这个登录表单，点提交" → 自动化表单操作
  • "抓取这个页面的数据" → 页面数据采集

案例 2：Sequential Thinking — 分步推理

让 Claude 在复杂问题上有条理地分步推理，支持回溯修正和分支探索。

配置：

claude mcp add sequential-thinking -- npx @anthropic/mcp-sequential-thinking@latest

完整参数：

参数说明
thought当前思考步骤的内容
nextThoughtNeeded是否需要继续下一步思考
thoughtNumber当前是第几步
totalThoughts预估总步数（可动态调整）
isRevision是否为对之前步骤的修正
revisesThought修正的是哪一步
branchFromThought从哪一步分支出去
branchId分支标识（同一问题可多分支并行）
needsMoreThoughts到达预设步数后是否还需要更多

引导示例：

# 简单分步
"用 sequential thinking 分析这个架构，分 5 步思考"

# 带分支的探索
"先分析方案 A 的优缺点（第 1-4 步），
 然后从第 2 步分支分析方案 B"

# 带修正的深度推理
"逐步推理这个问题，如果发现之前的某一
 步不对就回溯修正"

适用场景：复杂的架构设计、逻辑推导、边界模糊问题、多方案对比。

案例 3：Context7 — 实时文档查询

让 Claude 获取库/框架的实时文档，不再依赖训练数据。适合查最新的 API 用法。

配置：

claude mcp add context7 -- npx @anthropic/mcp-context7@latest

使用流程：

第1步：resolve-library-id("Python")     → /python/cpython
第2步：query-docs("Python 3.12 新特性") → 最新文档

应用场景：

  • "React 19 的 use() Hook 怎么用？" → 查最新 API
  • "Next.js App Router 怎么配中间件？" → 跳过旧版教程
  • "这个库的最新版本有什么变化？" → 对比文档

四、配合 MCP 工作的黄金法则

明确告诉 Claude 用哪个 MCP 工具。

✅ 好的写法：
"用 browser 打开 example.com，截图发给我"
"查一下 Context7 里 React 的 useTransition 文档"
❌ 差的写法：
"打开这个网页"（Claude 可能用 WebFetch 而非 Playwright）
"React 怎么用"（可能凭记忆回答，不够准）

MCP 工具会自动注册到 Claude 的工具列表中。如果 Claude 该用 MCP 却没用，直接说"用 XXX 来做"即可。

五、MCP vs Skills vs Hooks 的区别

MCPSkillsHooks

六、常见问题排查

现象原因解决
MCP 工具不出现未安装或配置错误claude mcp list 检查
MCP 工具超时服务卡住claude mcp restart <名称>
配置文件不生效JSON 格式错误检查 .mcp.json 语法
Stdio 模式报错Node.js 版本不够确认 Node.js ≥ 18
WebFetch 报 "无法验证域名"网络代理阻断验证settings.json 加 "skipWebFetchPreflight": true
