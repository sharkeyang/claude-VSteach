第 14 课 · Superpowers — 把 Harness 工程自动化的插件

前两课我们讲了 Harness 工程的 5 条核心方法。这一课介绍一个能自动执行这些方法的插件——Superpowers。

Superpowers 是 Anthropic 官方插件市场收录的开源方法论插件，由 Prime Radiant 团队维护。它不是教你"应该怎么做"，而是给 AI 一套硬性规则，让 AI 自动按 Harness 方法工作。

✅ 本环境已安装： /plugin install superpowers@claude-plugins-official 安装位置：~/.claude/plugins/cache/claude-plugins-official/superpowers/6.1.0/

一、Superpowers 是怎么工作的

Superpowers 的核心机制是：Skill 自动触发。你不需要手动输入 /skill-name，AI 在对话开始时会自动检查所有 Skill，发现适用的就自动执行。

它的源代码里写死了这条规则：

"如果你觉得有哪怕 1% 的可能某个 Skill 适用，你绝对必须调用它。
如果一个 Skill 适用于你的任务，你没有选择权。你必须使用它。
这是不可谈判的。你不能找借口绕过它。"

这意味着：你提出需求 → AI 自动触发 brainstorming（先方案）→ writing-plans（拆解任务）→ subagent-driven-development（多 Agent 实现）→ verification-before-completion（验证）→ finishing-a-development-branch（提交）——一条龙自动完成。

三、每条 Harness 方法如何被 Superpowers 落地

方法 ② 先方案后编码 → brainstorming + writing-plans

brainstorming 源代码中写了一个硬性门禁（HARD GATE）：

"在提交设计并获得用户批准之前，不允许调用任何实现技能、写代码、搭建项目。
这适用于每个项目，无论看起来多简单。"

它甚至明确列出了反模式：

❌ "这太简单了不需要设计" — 这是反模式。
简单项目正是隐藏假设导致最多返工的地方。
设计可以很短（几行字），但你必须展示并让用户批准。

完整流程：

探索项目上下文 → 逐个提问澄清需求
→ 提出 2-3 个方案（含推荐）
→ 分块展示设计 → 用户确认
→ 写设计文档并提交
→ 用户审查文档
→ 只有确认后，才能调用 writing-plans 进入编码

设计批准后，writing-plans 将需求拆解为 2-5 分钟/个 的可执行任务，每个任务包含精确的文件路径、完整代码和验证步骤。

方法 ③ MCP + Skills → using-superpowers

对话开始时，AI 在回答之前必须先检查所有可用的 Skill，如果有适用的就自动执行。这确保你安装的所有 Skill 和 MCP 都会被用上，不会闲置。

💡 互补关系：MCP 提供工具（浏览器、查文档、分步推理），Superpowers 确保 AI 会正确使用这些工具。两者搭配效果最好。

方法 ④ 自动测试验证 → test-driven-development + verification-before-completion

TDD 的铁律：

"没有失败的测试，就不能写生产代码。
先写了代码再写测试？删掉。重新开始。
没有例外。"

verification-before-completion 的铁律：

"没有新鲜的验证证据，就不能声称完成。
跳过任何一步 = 在撒谎，不是在验证。"

执行流程：

1. 识别：什么命令能证明这个声称？
2. 运行：执行完整命令
3. 读取：完整输出，检查退出码
4. 验证：输出是否确认了声称？
   - 否 → 用证据说明实际状态
   - 是 → 用证据一起声明结果

红旗警示（看到这些要警惕）：

  • "应该能通过"、"大概没问题"、"看起来可以"
  • 在验证之前就表达满意（"太棒了！""完美！"）
  • 相信 agent 的报告而不是自己验证

方法 ⑤ 文档+Git 存档 → finishing-a-development-branch

实现完成后的标准化收尾流程：

Step 1：验证测试 → 不通过就停下
Step 2：检测环境（普通 repo / worktree）
Step 3：确定基准分支
Step 4：展示 4 个选项：
   ① 合并到主分支
   ② 创建 PR
   ③ 保留分支
   ④ 丢弃改动
Step 5：执行选择 + 清理工作区

四、多 Agent 协作

方案 A：subagent-driven-development（推荐）

主会话内启动子代理，一个实现、一个审查，循环推进：

读取 plan → 创建任务列表
→ 循环：
  ① 启动 implementer 子代理（独立上下文）
  ② implementer 实现、测试、提交、自审
  ③ 启动 task reviewer 子代理审查
  ④ 有 critical 问题 → 启动 fix 子代理修复
  ⑤ 通过 → 标记任务完成
→ 全部完成后 → 启动最终代码审查
→ 调用 finishing-a-development-branch 收尾

方案 B：dispatching-parallel-agents（独立并行）

适用于互不相关的任务同时进行：

有多个独立问题？
  ├─ 有关联 → 一个 agent 全部处理
  └─ 无关联 → 可以并行？
               ├─ 是 → 并行派发（每个问题一个 agent）
               └─ 否 → 串行 agent

每个 agent 获得独立的上下文，互不干扰，也不消耗主会话的 Context。

五、安装与使用

安装

# 从官方插件市场安装
/plugin install superpowers@claude-plugins-official

# 从 Superpowers 自有市场安装（备选）
/plugin marketplace add obra/superpowers-marketplace
/plugin install superpowers@superpowers-marketplace

# 重载使生效
/reload-plugins

使用方式

什么都不用做。 Superpowers 的 Skill 在对话中自动触发。你只需要像平常一样提出需求：

你："帮我做一个股票分析工具"
→ brainstorming 自动启动：追问需求、输出设计方案
→ 你确认后：writing-plans 自动拆解任务
→ subagent-driven-development 自动调度多个 agent 实现
→ 完成后：自动测试验证 + 提交代码
