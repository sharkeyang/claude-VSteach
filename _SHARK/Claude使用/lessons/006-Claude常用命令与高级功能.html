第 11 课 · Claude 常用命令与高级功能

Claude Code 提供了很多命令和功能，有些天天用，有些你可能还没接触过。本课覆盖全部常用命令，重点讲解 subagent、workflow、plan 模式等你不熟悉但极其实用的高级功能。

一、基础命令（日常高频）

/help 高频
显示所有可用命令的帮助信息。任何时候忘了命令，先打这个。
/help → 列出全部命令
/help review → 查看 review 命令的详细帮助
/clear 高频
清空当前对话历史。当上下文太乱、Claude 开始"忘记"前面内容时使用。
/clear → 清空对话，重新开始
/config 高频
查看或修改当前配置（主题、模型、语言等）。
/config → 查看当前配置
/fast 高频
切换快速模式。开启后 Claude 输出更快（但质量略有降低），适合简单任务。
/fast → 开启/关闭快速模式
/init 高频
扫描项目代码，自动生成 CLAUDE.md 规则文件。新项目启动的第一步。
/init → 为当前项目生成 CLAUDE.md

二、计划与审查命令（先方案后编码）

/plan 🔥 强力
编码前先出方案。Claude 进入"计划模式"，只分析设计不出代码，你确认后再实施。
/plan 帮我加一个从AKshare下载可转债数据的功能
→ Claude 输出：要改哪些文件、改什么、影响范围、风险
→ 你确认后，再让它动手写代码
💡 /plan 什么时候用？
  • 要加一个新功能，不确定从哪里下手
  • 要改多个文件，怕 Claude 改漏或改错
  • 有几种实现方式，想让 Claude 先分析优劣
  • 项目比较大，不想让 Claude 直接"一把梭"
这是 Harness 方法论中"先方案后编码"的最直接实现。
/review 🔥 强力
审查当前代码改动（diff），自动检测 Bug、性能问题、代码质量问题。
# 审查当前所有改动
/review

# 审查特定文件
/review zdata.py
/code-review 🔥 强力
更详细的代码审查，包含修复建议。可以直接让 Claude 修复发现的问题。
/code-review --fix → 审查并自动修复问题
/code-review --comment → 审查并以注释形式输出
/verify ⚡ 新功能
代码改完后自动运行测试、Lint 检查，确认功能正常。打通"写完→验证"的闭环。
/verify → 自动运行验证流程
/security-review
安全审查，检查代码中的安全漏洞和风险点。
/security-review → 审查当前改动中的安全问题

三、Agent / Subagent（核心高级功能）🚀

这是 Claude Code 最强大的功能之一，值得花时间理解。

什么是 Subagent？

Subagent（子智能体）就是"让 Claude 再开一个 Claude"——它可以独立去完成一个任务，同时你继续和主 Claude 聊别的事。

Agent（子智能体）🔥🔥 极强力⚡ 建议掌握
在对话中直接启动一个或多个子智能体，每个子智能体独立工作，互不干扰。
# 启动一个子智能体处理特定任务
@agent 帮我搜索这个文件夹里所有过期的备份文件

# 多个子智能体并行工作（同时启动）
帮我启动两个 agent：
第一个搜索 Python 文件中的 TODO
第二个统计每个目录的文件数量

Subagent 的核心使用场景

场景不用 Subagent用了 Subagent
多文件搜索等 Claude 一个一个搜同时派多个 agent 分目录搜
代码审查 + 修复先审查，再看结果，再修复一个 agent 审查，一个 agent 修复
独立调研中断当前工作去查资料派 agent 去查，你先继续手上的事
批量操作手动一个个处理派 N 个 agent 并行处理
🚀 实战案例： 假设你要迁移 10 个 Python 脚本，修改其中的 import 路径：
@agent 改 _CHRIS/ 下第1-3个文件的 import
@agent 改 _CHRIS/ 下第4-6个文件的 import
@agent 改 _CHRIS/ 下第7-10个文件的 import
三个 agent 同时工作，总时间 = 改一个文件的时间，不是改十个文件的时间。

Subagent 的隔离模式

模式说明适用场景
普通在主仓库目录下工作搜索、分析、读文件
worktree在独立的 git worktree 中工作多个 agent 同时修改文件，避免冲突
remote在远程环境中工作需要独立计算资源或隔离环境

Subagent 的类型

类型擅长典型用途
general-purpose通用任务默认类型，啥都能干
Explore搜索查找在大量文件中找信息
Plan架构设计出方案、评估技术选型
claude-code-guideClaude 相关问题回答关于 Claude 功能的问题

四、Workflow（多智能体编排）🔧

Workflow（工作流）🔥🔥 极强力⚡ 建议掌握
比 Subagent 更高级的编排方式——把多个 agent 组织成流水线，一个 agent 的输出成为下一个 agent 的输入。
适用场景：
1. 审查代码 → agent 1 查语法错误 → agent 2 查性能问题 → agent 3 汇总报告
2. 迁移项目 → agent 1-5 并行迁移 → agent 6 做完整性检查
3. 多角度审查 → 同时启动 3 个 agent，分别从安全/性能/可维护性角度审查
💡 Subagent vs Workflow：
  • Subagent = 派一个人去干活。适合独立任务、并行搜索
  • Workflow = 组织一条流水线。适合多步骤、多角色协作的复杂任务
  • 怎么选：简单的并行任务用 Subagent，需要编排的复杂任务用 Workflow

五、循环与定时任务

/loop ⚡ 建议掌握
让 Claude 每隔一段时间自动重复执行某个任务。适合监控、轮询、定期检查等场景。
/loop 5m /review → 每 5 分钟审查一次代码改动
/loop /check-deploy → 每隔 10 分钟（默认）检查部署状态

实际应用：
"帮我定时检查数据下载是否完成，每 5 分钟检查一次"
→ Claude 会每隔 5 分钟检查一次，下载完成时通知你
/schedule ⚡ 建议掌握
安排定时任务，在指定时间自动执行。可以用 cron 表达式精确控制时间。
/schedule 每天9点帮我检查数据下载状态
/schedule 每周一早上生成上周总结

六、Skills 与 MCP 相关

/[skill-name] 高频
调用已注册的 Skill。Skill 的斜杠命令由 SKILL.md 中的 name 字段定义。
/chris-daily-plan → 生成 Chris 每日计划
/calligraphy 陋室铭 → 生成陋室金字帖
直接说"生成每日计划"也会自动匹配
/mcp ⚡ 建议掌握
查看、管理 MCP 服务器连接状态。
/mcp → 列出所有已配置的 MCP 服务器
/mcp restart filesystem → 重启 filesystem MCP

最重要的三件事

  1. 先 /plan 再编码 — 这是 Harness 方法论的核心，避免做无用功
  2. 善用 @agent 并行 — 需要同时做多件事时，派 agent 远比一个一个做快
  3. 写完 /review + /verify — 不验证不交付，这是质量底线
🎯 一句话总结：
/plan 保方向 → @agent 提效率 → /review + /verify 保质量
